# Source Mockups

This folder contains the **approved full-section Paper Tiger mockups**.

These mockups are the **primary visual authority** for implementation. They define:

- section composition;
- scale and placement of text;
- relationship between copy and illustration;
- negative space;
- page rhythm;
- the overall Paper Tiger feeling.

## Critical implementation rule

These files are **not** production section images to be pasted into the website as flat screens.

They are also **not** a source for blind automatic asset slicing.

The live website must be reconstructed as:

- real HTML typography;
- real responsive layout;
- real cards and forms;
- selectively prepared image assets only where they help reproduce the approved composition faithfully.

## How to use the mockups correctly

Use the mockups to:

- match each section visually, not just semantically;
- recreate hierarchy, spacing, and overall balance;
- decide whether a section needs:
  - one bounded illustration composition;
  - a small number of meaningful grouped assets;
  - or a rare isolated asset for a clearly separate object;
- preserve the approved desktop composition before adapting for mobile.

## What not to do

Do **not**:

- stack these mockups as full-width PNG sections;
- algorithmically crop dozens of fragments and scatter them across the page without checking the visual result;
- generate assets from coordinate guesses alone;
- replace the composition with a generic landing-page layout;
- treat the mockups as optional inspiration rather than the actual visual target.

Any derived artwork must be visually checked against the source mockup and used only if it improves fidelity.
